# 奶奶来都能看懂：把复杂技术讲成直觉

这个栏目专门把“听起来很难”的技术，翻译成“能想象、能复述、能落地”的语言。
不堆术语，不炫概念，先建立直觉，再补工程细节。

---

## 写作原则

- 先说人话版定义，再上技术术语。
- 先讲“为什么会有这个问题”，再讲“怎么解决”。
- 每篇尽量给一个贴近业务的小例子。
- 段落短、要点化，便于快速复习和转述。

---

## 目录地图

| 模块 | 你会学到什么 | 适合谁 |
| --- | --- | --- |
| 基础 | 计算机常识、系统运行直觉 | 初学者/转岗同学 |
| 大模型 | LLM 基础概念、工程认知、工具协作 | 想做 AI 应用的开发者 |
| 12因素智能体 | 从 Demo 到生产的智能体工程方法 | 需要落地 Agent 的同学 |
| Neo4j | 图数据库的核心模型与使用场景 | 需要关系建模的后端工程师 |

---

## 推荐阅读顺序

1. 基础：先把系统直觉补齐。
2. 大模型：建立 AI 时代的软件认知。
3. 12因素智能体：进入可上线的工程方法。
4. Neo4j：按需补充图数据库能力。

---

## 怎么使用这套内容

- 面向学习：按顺序看，优先理解核心比喻。
- 面向面试：重点看每篇的“结论 + 例子 + 边界”。
- 面向实战：把“奶奶版解释”翻译成你项目里的设计决策。

---

## 小结

真正理解一个技术的标志，是你能把它讲给非技术同学听。
如果奶奶都听懂了，面试官通常也会点头。
