# 因素10：小而专注的智能体

> 来源：12-Factor Agents / Factor 10（Small, Focused Agents）

## 一句话先懂

不要追求一个“全能大脑”，要建设一组“专科智能体”，每个只解决一个清晰问题。

## 奶奶版理解

看病时你不会找一个医生同时看牙、做眼科手术、做心脏手术。  
专科分工更靠谱。

## 原文核心逻辑

任务越大、步骤越多、上下文越长，模型越容易迷失。  
把任务边界收窄，通常能显著提高成功率和稳定性。

## 典型架构

- 路由器：判断任务类型
- 专科 Agent：在小范围完成高质量执行
- 编排层：聚合结果，必要时接力调用

## 如何划分边界（实操建议）

1. 按业务域拆：发布、工单、账单、客服。
2. 按风险拆：高风险动作单独 agent。
3. 按上下文拆：避免单 agent 持有过多异构信息。
4. 每个 agent 目标步数控制在可观测范围（如 3-10 步）。

## 常见误区

- 过度拆分，形成“微 agent 地狱”。
- 边界不清，多个 agent 相互甩锅。
- 只拆分不统一观测，排障更难。

## 我的思考

即便未来模型更强，这条仍然成立，因为系统复杂度并不只来自模型能力，还来自权限、合规、协作和运维。

“先小后大、逐步扩边界”是更稳的演进路径。

## 小结

- 小而专注提升可靠性。
- 拆分要按业务边界，不是按想象。
- 先把单域做到极致，再谈全局智能。
