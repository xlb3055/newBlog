# 12因素智能体（奶奶版）：从“能跑”到“可运营”

这套内容源自 HumanLayer 的 **12-Factor Agents**，我用更直白的中文重新组织成实战教程。
你可以把它理解为一条完整路线：先让智能体做对事，再让它稳定跑，最后让它可扩展、可维护。

---

## 你将解决的 3 类核心问题

- **做对事**：提示词、上下文、工具调用如何可靠。
- **跑得稳**：状态管理、控制流、错误恢复如何可控。
- **扩得开**：无状态设计、多入口触发、架构演进如何落地。

---

## 三阶段学习地图

### 阶段一：先让智能体“做对”

- 因素01 自然语言到工具调用
- 因素02 掌控提示词
- 因素03 掌控上下文窗口
- 因素04 工具本质是结构化输出

### 阶段二：再让智能体“跑稳”

- 因素05 统一执行状态与业务状态
- 因素06 用简单 API 实现启动/暂停/恢复
- 因素07 用工具调用联系人工
- 因素08 掌控控制流
- 因素09 把错误压缩进上下文

### 阶段三：最后让智能体“可规模化”

- 因素10 小而专注的智能体
- 因素11 从任何入口触发智能体
- 因素12 让智能体成为无状态归约器

---

## 时间紧张时的最小阅读顺序

1. `02` + `03`：提示词与上下文（决定上限）
2. `05` + `08` + `09`：状态、流程、错误（决定稳定性）
3. `10` + `12`：架构升级（决定可扩展性）

## 学完后的实践建议

- 每读完一篇，立刻把一个想法改造成“可执行步骤”。
- 每完成三篇，做一次小复盘：问题、方案、踩坑、改进。
- 别追求“大而全”Agent，先做“小而准”的任务闭环。
