# 技术提升：把“知道”变成“能落地”

这里不讲空概念，专门讨论线上最容易出问题的工程场景：并发、分布式、可用性、性能与容错。
目标只有一个：让你在面试和真实项目里都能给出“能执行、能解释、能权衡”的方案。

---

## 你会获得什么

- 看懂场景背后的核心矛盾：一致性、吞吐、延迟、成本。
- 学会从约束条件推导方案，而不是背固定答案。
- 知道每种方案的边界：什么时候该用，什么时候别用。
- 能输出“设计思路 + 关键实现 + 风险兜底”的完整回答。

---

## 场景地图（按问题类型）

| 问题类型 | 典型主题 | 你会关注的核心点 |
| --- | --- | --- |
| 服务通信 | RPC | 协议、序列化、超时重试、服务治理 |
| 异步解耦 | MQ、消费模式 | 顺序性、幂等、重试、死信与积压 |
| 分布式互斥 | 分布式锁、分布式单例 | 锁安全、可用性、过期与误删 |
| 高并发治理 | 秒杀、并发流统计 | 削峰限流、热点隔离、数据一致性 |
| 业务基础设施 | 短链接 | ID 生成、路由、缓存、回源 |
| 安全与合规 | 拦截与脱敏 | 规则匹配、性能、误判漏判 |
| 研发效率 | 外网访问内网服务 | 安全访问、隧道稳定性、权限边界 |

## 推荐阅读顺序（从基础到复杂）

1. `Rpc.md`：先建立服务通信与超时重试的基础认知。
2. `mq.md` + `mq消费方式.md`：理解异步系统的可靠性设计。
3. `分布式锁.md` + `分布式单例.md`：掌握跨节点协调。
4. `秒杀.md` + `并发流统计.md`：进入高并发实战。
5. `短链接.md` + `拦截器和dfa算法脱敏.md`：补齐基础设施能力。
6. `外网开发访问内网服务.md`：完善工程协作与运维场景。

---

## 每篇文章建议这样读

- 先读“业务约束”：QPS、延迟目标、可用性等级。
- 再看“方案对比”：为什么选 A，不选 B。
- 最后看“失败路径”：超时、重试、降级、补偿怎么做。

---

## 小结

工程场景题真正考察的是“判断力”。
把每篇文章当作一次架构演练：先识别矛盾，再做取舍，最后给兜底。
