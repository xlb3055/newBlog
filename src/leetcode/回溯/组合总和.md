# 39. 组合总和

---

## 题目
**描述：**
给你一个无重复元素的整数数组 `candidates` 和一个目标整数 `target`，找出 `candidates` 中可以使数字和为目标数 `target` 的所有不同组合。
`candidates` 中的同一个数字可以无限制重复被选取。如果至少一个数字的被选数量不同，则两种组合是不同的。

对于给定的输入，保证和为 `target` 的不同组合数少于 150 个。

**示例：**
示例 1:
```
输入: candidates = [2,3,6,7], target = 7
输出: [[2,2,3],[7]]
解释: 2 和 3 可以形成一组候选，2 + 2 + 3 = 7 。注意 2 可以使用多次。
7 也是一个候选，7 = 7 。
```
示例 2:
```
输入: candidates = [2,3,5], target = 8
输出: [[2,2,2,2],[2,3,3],[3,5]]
```
示例 3:
```
输入: candidates = [2], target = 1
输出: []
```

---

## 解题思路
### 核心方法
**回溯法**：
1.  **路径选择**：维护一个临时列表 `temp`，存储当前已选择的数字。
2.  **递归终止条件**：
    - 若当前剩余目标值 `target < 0`，直接返回（路径无效）。
    - 若当前剩余目标值 `target == 0`，将当前临时列表加入结果列表（路径有效）。
3.  **回溯过程**：从 `start` 开始遍历数组，将当前数字加入临时列表，递归进入下一层（剩余目标值减去当前数字，且保持 `start` 不变以允许重复选取）；递归返回后，将该数字从临时列表中移除，继续尝试其他数字。
4.  **避免重复组合**：通过控制遍历的起始位置（从 `start` 开始），确保组合按非降序生成，从而避免重复的组合（如 `[2,3]` 和 `[3,2]` 被视为同一组合）。

## 代码实现

### 回溯法（Java）
```java
import java.util.*;

class Solution {
    List<List<Integer>> ans = new ArrayList<>();
    List<Integer> temp = new ArrayList<>();
    
    public List<List<Integer>> combinationSum(int[] candidates, int target) {
        dfs(candidates, target, 0);
        return ans;
    }
    
    public void dfs(int[] candidates, int target, int start) {
        if (target < 0) return;
        if (target == 0) {
            ans.add(new ArrayList<>(temp));
            return;
        }
        for (int i = start; i < candidates.length; i++) {
            temp.add(candidates[i]);
            dfs(candidates, target - candidates[i], i);
            temp.remove(temp.size() - 1);
        }
    }
}
```

## 复杂度分析
- **时间复杂度**：`O(S)`，其中 `S` 是所有满足条件的组合的长度之和。在最坏情况下，每个组合的长度为 `target / min(candidates)`，而组合的数量取决于输入的具体情况。
- **空间复杂度**：`O(target / min(candidates))`，递归栈的深度和临时列表的空间均取决于目标值与最小候选数的比值。

---

## 总结
1.  回溯法是解决组合总和问题的经典方法，通过控制起始位置允许重复选取，并避免生成重复的组合。
2.  该方法的核心在于递归过程中保持 `start` 不变，从而允许同一数字被多次选取。
3.  适用于无重复元素且允许重复选取的组合求和问题，若数组包含重复元素或不允许重复选取，需要额外的去重或调整起始位置的逻辑。
