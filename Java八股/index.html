面试专题（Java 八股）：构建可复述的知识网络

这部分不是“背诵词条库”，而是面向后端面试的系统化复盘。 目标是让你在高频问题上做到三件事：说得准、讲得透、落得下。

你会在这里解决什么问题

  • 概念知道但讲不清原理。
  • 原理知道但说不出业务场景。
  • 场景能说但无法给出工程取舍。

模块地图

模块关注重点面试高频问法
Java 基础语法细节、JVM 相关机制、常见陷阱值传递、反射、泛型、异常体系
Java 并发线程模型、锁、JMM、并发容器synchronized vs Lock、AQS、线程池
Java 集合常用集合实现与源码思路HashMap 扩容、并发安全、选型依据
计算机基础操作系统、网络、Linux进程线程、TCP、IO、多路复用
数据库MySQL、Redis、MongoDB、ES索引、事务、缓存一致性、慢查询
杂碎记录零散高频点与易错补充快问快答、细节反杀题

推荐复习顺序（从底层到系统）

  1. Java 基础 + Java 集合
  2. Java 并发
  3. 操作系统 + 网络 + Linux
  4. MySQL + Redis（再扩展到 MongoDB / ES）
  5. 杂碎记录查漏补缺

每篇建议用这 5 步复盘

  1. 定义一句话：这个概念到底是什么。
  2. 原理三要点：核心机制、关键流程、性能特征。
  3. 场景举例：在真实业务里用在哪里。
  4. 风险边界：常见误用和坑点。
  5. 对比选型：和替代方案相比为什么这样选。

面试回答结构（可直接套用）

  • 先给结论：一句话回答问题本质。
  • 再讲原理：按“数据结构/流程/复杂度”展开。
  • 接着给场景：结合订单、库存、缓存等业务。
  • 最后说取舍：优点、缺点、适用边界与监控兜底。

小结

八股真正的价值，不是“背过”，而是“能在不同问题里迁移使用”。 把知识点连成网，面试时你就不会被追问打散。
