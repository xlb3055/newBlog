# AGENTS.md - LeetCode 题解规范

## 适用范围
- 适用于 `src/leetcode/**` 下的题解文章。

## 目录与命名
- 目录：`src/leetcode/<分类>/`（例如 数组/哈希/贪心/堆）。
- 文件名：`<题目中文名>.md`，保持简洁，不加空格。

## Front Matter 规范
每篇文章必须包含 YAML Front Matter，字段结构如下：
```yaml
---
icon: pen-to-square
date: YYYY-M-D
category:
- 后端
tag:
- 题目分类（如 数组/哈希/贪心/堆/优先队列）
- leetcode
---
```
- `date` 按实际日期填写。
- `tag` 至少包含题目分类和 `leetcode`。

## 推荐结构（从上到下按顺序）
1. 标题（H1）：`<题号>. <题目名>`，例如「1. 两数之和」。
2. 题目描述：简洁说明题意与限制条件。
3. 示例：至少 1-2 组，包含输入/输出/解释。
4. 解题思路：说明核心思想与关键步骤（如 DP/贪心/双指针/堆）。
5. 代码实现：给出可运行的 Java 版本。
6. 复杂度分析：时间复杂度 + 空间复杂度。
7. 总结：用 2-4 条要点回收解法与注意点。

## 写作风格
- 先题意后思路，再给代码与复杂度。
- 解释关注为什么这样做，而不是重复题干。
- 文字简洁、条目化，避免过长段落。

## 代码示例规范
- 默认语言：Java（除非用户明确指定其他语言）。
- 代码块必须标注语言，例如 ```java。
- 推荐结构：`Main` 负责输入输出，`Solution` 负责算法实现。
- 代码尽量可直接运行；若输入格式简化，需说明简化点。

## 最终检查清单
- Front Matter 完整且字段正确。
- 标题格式正确，包含题号与题名。
- 至少一个示例，且输入/输出明确。
- 思路描述清晰，有关键步骤。
- 代码可运行，语言标注正确。
- 明确给出时间与空间复杂度。
